Movies API Challenge (Step-by-Step in Postman)

1) Start the server
- Open terminal in: Not Quiz/movies-app/server
- Run: npm start
- Default URL: http://localhost:3001
- If 3001 is busy, change APP_PORT in .env to 3002, then restart.

2) Create a Postman collection
- Open Postman.
- Click New Collection.
- Name it: Movies API.
- Add the requests below to this collection.

3) Create movie (POST /movie)
- Method: POST
- URL: http://localhost:3001/movie
- Body: raw + JSON
- Paste:
{
  "title": "Inception",
  "synopsis": "A thief enters dreams to steal secrets.",
  "genre": "Science Fiction",
  "director": "Christopher Nolan",
  "releaseDate": "2010-07-16",
  "rating": 8.8
}
- Click Send.
- Expected: 200 OK with created movie JSON (includes id).

4) Challenge test: duplicate title on create
- Send the same POST request again (same title).
- Expected: 400 Bad Request.
- Expected message: duplicate title exists.

5) Get all movies (GET /movie)
- Method: GET
- URL: http://localhost:3001/movie
- Click Send.
- Expected: list of movies ordered by title ascending (A to Z).

6) Challenge test: search by title or synopsis
- Method: GET
- URL: http://localhost:3001/movie?search=dream
- Click Send.
- Expected: returns movies where title OR synopsis contains "dream".

7) Challenge test: optional genre filter
- Method: GET
- URL: http://localhost:3001/movie?genre=Science
- Click Send.
- Expected: only movies whose genre matches "Science".

8) Challenge test: optional minRating filter
- Method: GET
- URL: http://localhost:3001/movie?minRating=8
- Click Send.
- Expected: only movies with rating >= 8.

- Invalid test:
  URL: http://localhost:3001/movie?minRating=abc
  Expected: 400 Bad Request.

9) Get movie by id (GET /movie/:id)
- Method: GET
- URL example: http://localhost:3001/movie/1
- Click Send.
- Expected:
  - 200 with movie data if id exists
  - 404 if id not found

10) Update movie (PUT /movie/:id)
- Method: PUT
- URL example: http://localhost:3001/movie/1
- Body: raw + JSON
- Paste:
{
  "title": "Inception Updated",
  "rating": 9.0
}
- Click Send.
- Expected: success message.

11) Challenge test: duplicate title on update
- Make sure you already have 2 different movies.
- Try updating movie B title to movie A title.
- Expected: 400 Bad Request (duplicate title).

12) Delete movie (DELETE /movie/:id)
- Method: DELETE
- URL example: http://localhost:3001/movie/1
- Click Send.
- Expected: delete success message.
- Send same DELETE again on same id.
- Expected: 404 (not found).
